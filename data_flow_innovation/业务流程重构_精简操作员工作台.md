# 业务流程重构：精简操作员工作台设计

> **核心目标**：去除 Chrome 插件依赖，去除任务大厅手工操作，基于现有 API 接口，
> 构建一个极简的单页操作员工作台。人工职责缩减为：**打开公司官网 → 找到链接 → 粘贴链接 → 确认**，
> 其余一切由系统自动完成。

---

## 一、现有流程痛点（操作层面）

```
现有操作步骤（单家公司 · 约 10-20 分钟）

  Step 1  登录系统任务大厅（浏览器）
  Step 2  浏览任务列表，手工领取任务
  Step 3  打开任务详情，记住需要收集的报告清单
  Step 4  打开 Chrome 插件，确认登录状态
  Step 5  根据记忆打开公司官网
  Step 6  在官网中寻找 IR 页面（可能多层导航）
  Step 7  找到年报/季报 PDF 链接
  Step 8  通过插件点击下载
  Step 9  等待上传 S3 完成
  Step 10 返回任务大厅，标记完成
  Step 11 重复 Step 3-10 遍历所有公司

  痛点：
    · 步骤分散在两个系统（任务大厅 + Chrome 插件）之间
    · 人工需记忆"要收集什么"再去找
    · 插件上传失败时需手工重试
    · 操作路径长，认知负荷高
    · 多窗口切换，容易出错和遗漏
```

---

## 二、重构后新流程（目标）

```
新操作步骤（单家公司 · 约 2-4 分钟）

  Step 1  打开工作台网页（已自动完成：鉴权 + 领取任务 + 展示待办）
  Step 2  点击公司官网链接（工作台直接提供）
  Step 3  在官网找到目标 PDF / 音频链接
  Step 4  将链接粘贴到工作台输入框，点击"提交"
  Step 5  工作台自动：下载 → 验证 → 上传 S3 → 回写状态
  Step 6  自动切换到下一家公司

  去除：Chrome 插件、任务大厅、手工领取、手工标记完成
  保留：人工找链接这一核心判断动作（AI 兜底时的最后防线）
```

### 3.2 新增核心接口设计：`POST /api/collect/submit-url`

```python
# 请求体
class SubmitURLRequest(BaseModel):
    task_id: int                    # 任务 ID
    company_id: str                 # orbit_entity_id
    report_type: str                # "annual_report" | "quarterly_report" | "earnings_audio"
    year: int                       # 报告年份
    quarter: Optional[int]          # 季度（季报专用，1-4）
    source_url: str                 # 操作员粘贴的原始链接
    comment: Optional[str]          # 补充备注（可选）

# 响应体
class SubmitURLResponse(BaseModel):
    status: str                     # "processing" | "success" | "failed"
    job_id: str                     # 异步任务 ID，前端轮询进度
    message: str

# 后端自动执行（操作员无感知）：
#   1. 解析 source_url（处理重定向、签名 URL 等）
#   2. 下载文件（带进度回调）
#   3. 内容验证（PDF 页数、文件类型、公司名匹配）
#   4. 上传至 AWS S3（路径：/{company_id}/{year}/{type}/filename）
#   5. 调用 Orbitfin API 回写 source_pages 记录
#   6. 更新本地状态，通知前端刷新
```

### 3.3 Token 管理重构

**现有问题**：依赖 Chrome DevTools Protocol (CDP) 读取 localStorage 中的 JWT，
要求操作员必须保持 Chrome 登录态，耦合度高。

**重构方案**：将 Token 管理提升为后端服务，操作员只需在工作台登录一次：

```
重构后 Token 生命周期：

  操作员登录工作台（标准表单登录）
       │
       ▼
  后端代理登录 Orbitfin（用操作员凭证换取 JWT + Refresh Token）
       │
       ▼
  Token 持久化到后端 Session（操作员浏览器只持有 Session Cookie）
       │
       ▼
  后端 Token 自动续期（RefreshToken 到期前静默刷新）
       │
       ▼
  操作员无需关注 Token，工作台始终可用

  优点：
    · 彻底去除 Chrome CDP 依赖
    · 去除 Chrome 插件
    · 支持多操作员同时工作（各自独立 Session）
    · Token 失效只需重新登录工作台，无需手动刷新浏览器
```

## 六、后端新增模块设计

### 6.1 文件采集服务 `CollectService`

```python
# backend/services/collect_service.py

class CollectService:
    """
    接收操作员提交的 URL，完整执行：
    下载 → 验证 → 上传 S3 → 回写 Orbitfin → 更新本地状态
    """

    async def submit_url(self, req: SubmitURLRequest, jwt_token: str) -> str:
        """返回 job_id，异步执行采集流程"""

    async def _download(self, url: str) -> tuple[bytes, str]:
        """下载文件，返回 (内容, 实际文件名)
        处理：30x 重定向、Content-Disposition 文件名提取
        """

    async def _validate(self, content: bytes, req: SubmitURLRequest) -> ValidationResult:
        """
        验证文件内容：
        PDF → 提取首页文字，LLM 确认报告类型、年份、公司名
        音频 → 检查 MIME type、时长 > 5分钟
        """

    async def _upload_s3(self, content: bytes, req: SubmitURLRequest, filename: str) -> str:
        """上传至 S3，返回 s3_path
        路径规则：{company_id}/{year}/{report_type}/{filename}
        """

    async def _writeback_orbitfin(self, req: SubmitURLRequest, s3_path: str, jwt_token: str):
        """调用 Orbitfin source_pages API 回写采集记录"""
```

### 6.2 Session-based Token 管理器

```python
# backend/services/token_manager.py

class TokenManager:
    """
    持久化管理操作员的 JWT + Refresh Token。
    替代现有的 Chrome CDP 方案。
    """

    def login(self, username: str, password: str) -> Session:
        """用操作员凭证换取 JWT，持久化到 Session"""

    def get_valid_token(self, session_id: str) -> str:
        """获取有效 JWT，必要时自动 refresh"""

    def _refresh(self, session_id: str) -> str:
        """调用 POST /prod/login/refresh 静默续期"""
```

## 八、新旧流程对比

### 操作步骤数量

| | 现有流程 | 新流程 |
|--|---------|--------|
| 登录操作 | 登录任务大厅 + 登录插件（2次） | 登录工作台（1次） |
| 领取任务 | 手动浏览 + 点击领取 | **自动**（无感知） |
| 查看待办 | 切换到任务详情页 | 工作台左侧随时可见 |
| 打开官网 | 手动复制 URL 打开 | **工作台直接提供链接，一键打开** |
| 提交链接 | 通过插件点击下载按钮 | 粘贴 URL + 点击提交（2步） |
| 上传确认 | 等待插件上传，手动确认 | **自动**，工作台实时反馈 |
| 标记完成 | 返回任务大厅手动标记 | **自动**（链接提交成功即标记） |
| 切换公司 | 手动在列表中找下一家 | **自动推进**到下一家 |

| | 现有流程 | 新流程 |

## 十、关键设计原则

1. **最小人工决策**：操作员唯一需要判断的是"这个链接对不对"，其他全部自动化。

2. **零切换成本**：工作台一个页面包含所有信息，操作员不需要在系统之间来回切换。

3. **失败友好**：下载失败、验证不通过时，给出明确原因和建议操作，而非让操作员猜测。

4. **最小改动现有代码**：复用已实现的 `/api/test/*` 接口，新增接口以 `/api/collect/*` 和 `/api/workspace/*` 命名空间隔离。

5. **可渐进增强**：Phase 1 就可以上线替代现有流程，AI 辅助预填是可选叠加层，不阻塞基础使用。
